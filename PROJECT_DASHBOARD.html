📊 Project Overview

Business Problem

Loan officers at SZE Bank spend significant time manually reviewing loan applications. This tedious process is:

  • Time-consuming - Manual review of hundreds of applications
  • Inconsistent - Subjective criteria across different reviewers
  • Error-prone - Opportunities for bias and human error
  • Expensive - Requires dedicated staff for administrative review

Solution

We developed a machine learning system that automates loan eligibility prediction using historical application data and outcomes. This solution provides:

🎯 Automation

Instant predictions on loan applications

📊 Consistency

Uniform decision criteria for all applications

📈 Efficiency

Process hundreds of applications daily

🔍 Transparency

Clear feature importance for decisions

🤖 Model Architecture

Algorithm Selection: XGBoost

XGBoost (Extreme Gradient Boosting) is a gradient-boosted decision tree ensemble that:

  • Handles both numeric and categorical data efficiently
  • Provides built-in feature importance scoring
  • Resistant to overfitting with proper regularization
  • Scales well to large datasets
  • Provides interpretable decision paths

Hyperparameters

eta (learning rate): 0.1 max_depth: 4 min_child_weight: 1 subsample: 0.8 (row subsampling) colsample_bytree: 0.8 (column subsampling) objective: binary:logistic eval_metric: error optimization: GridSearchCV (5-fold CV)

Data Pipeline

Stage Process Details
1. Loading Load raw data 491 training samples with 13 features
2. Preprocessing Handle missing values Mode and median imputation with business logic
3. Encoding One-hot encode categorical variables 7 categorical features → 14 numeric features
4. Splitting Train/validation/test split 70% train (343), 30% val (148), 8% test (123)
5. Training GridSearchCV optimization Hyperparameter tuning with cross-validation
6. Evaluation Performance metrics Accuracy, Precision, Recall, F1-Score, Confusion Matrix

Training Configuration

Optimization Method

GridSearchCV with 5-fold cross-validation for robust hyperparameter selection

Regularization

L1/L2 regularization and early stopping to prevent overfitting

Class Imbalance

Stratified sampling ensuring balanced representation in folds

Validation Strategy

Train on 70% data, validate on 30%, final test on unseen holdout set

Model Behavior Analysis

Conservative Approval Strategy:
The model has higher false positive rate (54.35%) than false negative rate (23.04%), showing a bias toward approval. This is strategically useful for loan decisions where:
  • Further human review can minimize risk of false approvals
  • Missing approvable loans (false negatives) can hurt bank competitiveness
  • The 81.37% recall ensures strong capture of eligible applicants

Decision Threshold

The model uses a default probability threshold of 0.5 for classification. This can be adjusted:

  • Lower threshold → More approvals (higher sensitivity, lower specificity)
  • Higher threshold → Fewer approvals (lower sensitivity, higher specificity)
  • Current balanced → 0.5 default threshold optimizes F1-score

Key Insights

  • Credit History Dominance: Previous credit behavior is the strongest predictor - 3x more important than total income
  • Loan Characteristics Matter: Term and amount decisions heavily influence approval likelihood
  • Income is Secondary: Total household income (applicant + co-applicant) accounts for ~19% combined importance
  • Geographic Impact: Property location (urban/semi-urban/rural) influences ~10.51% of decisions
  • Demographics Less Influential: Family status, education, and gender account for ~10% combined weight

🚀 Installation & Setup

Prerequisites

  • Python 3.8 or higher (tested on 3.11.13)
  • pip or conda package manager
  • 100MB free disk space
  • Git for version control

Step 1: Clone Repository

$ git clone https://github.com/yourusername/loan-approval-prediction-main.git $ cd loan-approval-prediction-main

Step 2: Create Virtual Environment

# On macOS/Linux $ python -m venv venv $ source venv/bin/activate # On Windows $ python -m venv venv $ venv\Scripts\activate

Step 3: Install Dependencies

$ pip install -r requirements.txt

Required Packages

Step 4: Run Web Application

$ python app.py # Application available at: http://localhost:5000

Step 5: Explore Jupyter Notebook

$ jupyter notebook # Open: notebook/Machine Learning Model Dev.ipynb

Troubleshooting

ModuleNotFoundError: Ensure all dependencies are installed with pip install -r requirements.txt

Port 5000 in use: Modify app.py to use different port or kill existing process

Kernel issues in Notebook: Restart kernel and run cells sequentially from top
